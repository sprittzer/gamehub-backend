Каталог игр — веб-приложение для сбора и систематизации информации о компьютерных играх. Пользователи могут просматривать каталог, искать игры по жанрам, платформам и годам выпуска, а также добавлять собственные оценки и отзывы. Разработка ведется с использованием FastAPI. Используется supabase.

# Sprint Review
По итогам первого спринта были успешно завершены следующие задачи:
- #1 — Настройка рабочего окружения и базовой конфигурации проекта
- #2 — Реализация модели и CRUD для сущности Game
- #3 — Создание REST API эндпоинтов для управления играми

Доставленный инкремент: К концу спринта мы имеем полностью рабочее бэкенд-приложение GameHub с настроенным окружением, созданной моделью данных "Игра" в базе данных и работающими REST API эндпоинтами (GET /games, GET /games/{id}, POST /games) с поддержкой пагинации и базовой фильтрации по жанру.

Не завершённые задачи:
- #7 — Реализация улучшенной фильтрации и поиска для списка игр (осталась в статусе in-progress)
- #8 — Написание базовых тестов для главных эндпоинтов (осталась в статусе todo)

Sprint Retrospective
Что прошло хорошо:
- Чёткое планирование и реалистичная оценка сложности задач (story points) позволили команде эффективно распределить усилия
- Последовательность выполнения задач (окружение → модель → API) создала прочный фундамент для дальнейшей разработки
- Использование GitHub Issues с метками sprint-1, todo/in-progress обеспечило прозрачность и наглядность прогресса

Что пошло не так:
- Задача по улучшенной фильтрации (#7) оказалась сложнее, чем ожидалось — реализация валидации query-параметров потребовала дополнительного времени
- Тестирование (#8) было отложено "на потом" и в итоге не было начато, что создаёт риск накопления технического долга
- Недостаточно детальное разбиение задачи #7 на подзадачи затруднило точную оценку времени её выполнения

Что можно улучшить в следующем спринте:
- Обязательно включать задачи по тестированию в план спринта, даже с минимальным объёмом (например, тесты для 1-2 ключевых эндпоинтов)
- Разбивать сложные задачи на более мелкие подзадачи перед планированием спринта для более точной оценки
